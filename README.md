# 云桌麻将 Godot 客户端

Godot 4.6.3 迁移版 Android 客户端。目标是替代原生 Android Canvas UI，使用开源游戏引擎做横屏牌桌、真实麻将牌图、轻量牌桌 UI 和后续动画/特效。

当前版本：`v1.0.162-godot`

## 当前功能

- 横屏 Godot UI。
- 复用 CC0 麻将牌 PNG、桌布、木纹、音效素材，并补齐 Godot 专用花牌 PNG。
- 单机模式：本地发牌、自动补花、三家 AI 自动行牌，三家电脑分别带有守门清、速攻和做大牌的稳定性格权重，出牌、防守、吃碰杠和自杠取舍会按座位风格变化；支持本地吃/碰/杠/胡、自摸、荒庄、包三搭包赔、清一色/混一色/碰碰胡/字一色/大三元/小三元/大四喜/小四喜/一条龙/断幺九/杠上开花/海底捞月/大吊车/抢杠胡等番型、多局分数保留、庄家连庄/下庄、暗杠/明杠/补杠、多人可选吃牌组合、胡牌优先于碰杠吃的响应优先级，以及基于向听数、有效进张、牌型潜力、对手副露威胁和防守风险的 AI 推荐出牌；AI 吃牌会逐一评估所有可吃组合，按综合收益选择具体吃法；AI 暗杠/补杠会综合向听、牌值、攻守压力、路线保护和抢杠风险后再决定是否执行。
- 联机模式：连接现有 `mahjong-room-server` JSON Lines 协议，支持创建房间、加入房间、开始、出牌、响应吃碰杠胡入口，联机吃牌可显示并发送具体吃法，兼容常见 room/game state 字段别名，并显示服务器确认、拒绝原因和房间日志。
- 牌局声音：启动、进桌、恢复前台、触摸/按钮/牌点击都会唤醒音频并重试 BGM；BGM 改用 44.1kHz 单声道 16-bit PCM WAV 音乐循环、禁用导入压缩并降低默认增益，避免手机端削波、重采样或压缩解码噪声；摸牌/出牌与碰杠胡音效复用持久播放器，快速行牌时减少节点分配和混音抖动；随包内置中文牌名和动作语音，出牌优先播放本地语音片段，不再依赖手机 TTS 初始化成功才有报牌；系统 TTS 作为兜底，设置面板提供“试音”一键重新开启并播放测试报牌。
- 设置面板：菜单和牌桌顶部可切换音乐、音效、报牌和快速模式，设置会保存到本机。
- 操作流畅度：菜单、按钮、行动区和手牌按下即执行；快速模式把 AI 计算等待和可见动作停顿分开，人出牌后、AI 杠牌后、每个 AI 出牌后都会留出短暂落桌间隔，让画面刷新、出牌音和报牌按顺序出现，避免三家 AI 声音堆到一起；BGM 已在播放时不再由后台看门狗定时重触发播放链路；帧循环只在下载、语音或联机场景进入对应轮询，联机 socket 轮询和下载进度刷新也按短时间窗节流，降低空闲帧和下载帧的主线程开销；人机对战中玩家侧 AI 辅助默认关闭，不再计算/显示推荐牌、最安牌、备选牌、风险徽标、提示按钮和响应建议；AI 连续行牌时整桌重绘合并到短刷新窗口，减少主线程卡顿；向听数、AI 出牌报告、牌序排序、牌面分类/显示名/语音名/颜色、十三幺牌面索引、UI 样式和对手威胁报告使用缓存，AI 推荐和进张候选只在收集完成后排序一次，重复牌候选先按牌面去重，每张候选弃牌只统计一次手牌计数并复用于向听、进张、形状质量、路线评分和路线报告，候选报告也可接收已扣减的手牌计数快照，常规出牌和副露后强制弃牌不再为每个候选重新 tile_counts，候选模拟复用一份工作手牌和工作计数并按牌面索引临时扣减/恢复，避免每个候选 duplicate 整手牌和计数快照；牌面计数数组统一用固定长度零填充 helper 初始化，减少高频手牌/公开牌计数里的解释器循环；出牌理由标签复用弃牌前原手牌计数快照判断七对保路线和孤张，避免每个候选再复制恢复原计数、重建原手牌数组和重新统计；AI/威胁报告缓存键使用稀疏牌面计数生成稳定牌序 key，避免每次查询都分配并填满 34 槽计数数组；AI 出牌报告写入缓存时只定向复制有效进张、剩余张数、喂牌详情和危险来源等可变字段，替代整份报告递归深拷贝；向听递归 memo key 复用单次紧凑计数键，紧凑计数键用字节缓冲生成常规 0-9 计数串，减少递归里重复小字符串数组分配；完整胡牌面子递归原地加减并使用局部 memo，避免每个分支复制计数数组；听牌价值评估在 AI 候选报告中复用已验证的有效待牌列表，跳过重复完整胡牌校验，并复用一个临时和牌手牌数组逐张 append/pop 计算待牌价值，不再为每个待牌 duplicate 手牌；同时把自家弃牌预先建成集合，判断回头待时不再反复扫描弃牌河；AI 候选弃牌一次分析番型路线、对子路线、十三幺、一条龙和听牌价值，再复用到路线评分、形状摘要和报告描述；吃碰杠响应先统计一次手牌计数，再用计数快照扣减响应牌并复用到向听、形状和路线收益，响应选项也复用同一份手牌计数判断胡/杠/碰/吃，避免同一响应反复复制手牌、count_tile 和扫描吃牌组合；同一张弃牌的 AI 吃碰杠响应会共享响应评估上下文，杠/碰/多种吃法复用手牌计数、公开牌计数、对手状态、压力上下文、风险向量和安全牌标签缓存；弃牌压力、喂牌风险、放铳风险和安全牌标签共享公开牌计数快照，筋线和壁牌安全判断直接用牌面索引、对手弃牌上下文和可见计数，不再构造锚点/壁牌临时数组，同一轮候选弃牌共享对手压力、听牌压力和威胁等级上下文，放铳总风险和最大危险来源合并为一次逐家扫描，逐家风险组件复用同一个结果字典，避免每个对手都分配组件报告，进张评估直接复用手牌计数并临时加减候选牌，常规候选出牌和副露后强制弃牌评估都共享公开牌计数、对手状态、风险向量和安全牌标签缓存，只需要风险分数的路径和威胁安牌候选排序直接读取风险向量分数，跳过中间摘要字典，威胁安牌候选只维护前 N 个轻量候选，避免为全量候选创建字典并整体排序，风险向量和喂牌报告缓存命中也只定向复制危险来源、喂牌详情等可变字段，缓存命中避免深拷贝嵌套报告，减少同一牌桌状态下提示、风险徽标、座位威胁和推荐的重复计算。
- 待牌价值性能：非听牌候选直接返回零值待牌指标，跳过攻击/待牌权重计算；AI 出牌候选报告对非听牌直接填充零值待牌字段，不再进入 `wait_value_metrics` 或读取自家弃牌 lookup，自家弃牌 lookup 也改为听牌候选按需生成；听牌候选复用 AI 出牌报告已计算的攻击倍率，避免同一候选重复读取分势和 AI 性格权重。
- AI 缓存键性能：弃牌压力、喂牌风险、放铳风险和安全牌标签在同一次上下文查询中复用已生成的缓存 key，避免命中检查和写入缓存时重复格式化同一字符串；4 张以内的牌面数组 key 使用局部索引排序直接生成，副露/小牌组缓存 key 不再构造计数字典和索引数组。
- 玩家辅助 UI 性能：推荐牌、最安牌、备选牌和高危确认复用 AI 出牌报告的定向复制，只保护有效进张、剩余张数、喂牌详情和危险来源等可变字段，避免 UI 刷新时对完整嵌套报告做递归深拷贝。
- 响应建议性能：吃牌选择和吃碰杠响应报告使用固定结构定向复制，只复制 `needed`、`meld` 等可变数组，避免响应建议和 AI 吃牌比选中对小型报告做递归深拷贝；玩家响应推荐候选懒创建并共享同一份 claim context，多种吃法/碰杠比选复用手牌计数、公开牌计数、压力上下文和风险缓存；吃牌候选生成直接按牌面索引检查两张所需牌，避免每个候选 duplicate/erase 顺子和构造需求计数字典；所需牌计数检查/扣减对 1-2 张牌走固定索引快速路径，常见吃牌响应不再构造临时需求字典；最佳吃法评分直接复用手牌计数快照并临时扣减/恢复所需牌，不再为每个吃法复制手牌数组后重新评估。
- 座位 UI 性能：牌桌重绘时每个对手只取一次威胁报告，座位徽标、颜色和威胁说明共用同一结果；同一次渲染中的三个对手面板共享 AI 评估上下文、风险向量和安全牌标签缓存；威胁报告缓存命中只做轻量保护性拷贝，减少手机端整桌重绘时的分配压力。
- 静态牌视图性能：手牌、弃牌、明刻、花牌和中心展示牌只渲染真实牌面图片，禁用代码绘制的牌面文字、角标、风险/推荐文字覆盖和文字兜底；弃牌等不可点击牌使用轻量 `Panel` 绘制，只为玩家可点击手牌保留按下即执行的按钮路径。
- 牌墙重绘性能：牌桌上的固定牌墙使用 4 个自绘 `WallBackStrip` 控件批量绘制牌背，不再为 56 个牌背逐个创建 `Panel` 节点；单个牌背组件仍保留完整纹理和标记用于需要细节展示的场景。
- 实时语音：联机牌桌内可开关麦克风，客户端以 `voiceMessage` 发送 PCM16/base64 音频分片并播放收到的语音。
- 更新模式：先读取 Godot APK 更新 manifest，版本较新时下载对应版本固定 APK 直链，并在下载完成后校验 SHA-256；manifest 不可用时回退当前版本固定 APK 直链。
- 轻量头像：牌桌座位使用 2D 玩家标识和行牌高亮，避免手机端每次重绘创建 3D `SubViewport` 造成卡顿。
- 牌势面板：牌桌顶栏显示四家比分和 AI 风格短标，中心盘集中显示余牌、上张、风位和当前状态；座位卡直接显示活动高亮、威胁徽标和 AI 性格。面板展示推荐出牌、具体听牌/进张候选及每张剩余张数、待牌薄厚、回头待提示、听牌价值番分、切孤张/保路线/防守/保好形等弃牌理由、两面/坎张/边张/对子形状质量、清一色/混一色/碰碰胡/字一色主路线、整场分势策略、攻守模式、风险、全现物/主威胁现物/熟张/筋线/壁牌安全性和当前最明显的防守威胁；AI 会结合弃牌数、副露数、余牌和牌路集中度估算对手快进/近听/疑听压力，并影响放铳风险与防守权重；对手座位会直接显示高/危威胁、疑听/近听徽标和偏向门类，便于扫桌读牌；高压局面行动区会直接提供“最安牌”一键出牌，手牌托盘、提示和牌势面板也会显示最安全弃牌；行动区提供“推荐牌”和低风险“备选牌”一键出牌按钮，“提示”会解释推荐原因、备选牌、防守重点和高危弃牌主要来源；高危弃牌确认时行动区会给出可直接点击的安全改打按钮；响应吃碰杠胡时会给出建议动作和理由，并在行动区标出建议吃碰杠胡或建议放弃；行动区新增当前意图提示条，手牌分组间隔显示万/条/筒/字/花标记，牌势卡片使用信号条强化推荐、收益和防守层级；结算面板显示每位玩家本局分数变化。
- AI 防喂牌：出牌评估加入公开信息喂牌风险，结合下家可吃、中张、生张、对手副露和弃牌偏向估算“喂吃/喂碰”压力；提示和高危确认会直接显示“喂某家吃/碰”等来源。
- 特殊牌型：七对、十三幺、一条龙、断幺九、大小三元和大小四喜纳入胡牌、向听、有效进张、计分与 AI 路线提示；六对重手会显示“路线七对”并优先保留对子切孤张，也会拒绝破坏闭门七对路线的低收益吃碰杠；十三幺高密度手牌会显示“路线十三幺”，一条龙高进度手牌会显示“路线一条龙”，断幺九高进度手牌会显示“路线断幺九”，三元/四喜重手会显示对应路线，AI 会优先保留目标路线牌并切非路线牌。
- Android APK 导出：release 已签名。

## 构建

```bash
cd /root/yunzhuo-mahjong-godot
GODOT_SILENCE_ROOT_WARNING=1 godot --headless --path . --export-release Android build/YunzhuoMahjongGodot-v1.0.162-godot.apk
```

Godot Android 设置：

- Java SDK Path: `/usr/lib/jvm/java-21-openjdk-arm64`
- Android SDK Path: `/opt/android-sdk`
- Export templates: `/root/.local/share/godot/export_templates/4.6.3.stable`
