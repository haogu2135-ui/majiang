# 性能优化测试指南

## 快速测试

### 1. 启动游戏并运行一局

```bash
cd /root/yunzhuo-mahjong-godot
godot --headless scripts/main.gd
```

或者在Godot编辑器中运行项目。

### 2. 观察性能改善

**UI响应速度：**
- 出牌后界面刷新应该明显更快
- 不应该感觉到AI"思考"时的卡顿
- 手牌点击后立即响应

**AI行为：**
- AI出牌决策时间正常（由于缓存，第二次相似决策会更快）
- AI推荐（如果启用）会在主界面显示后稍微延迟出现

### 3. 查看性能统计

在游戏代码中添加性能日志调用：

```gdscript
# 在适当位置（例如每局结束）添加
log_performance_stats()
reset_performance_stats()
```

**期望输出：**
```
=== 性能统计 ===
渲染: 150次, 平均35.2ms
AI决策: 48次, 平均220.5ms
向听缓存: 92.3% (1850/2004)
进张缓存: 73.8% (885/1199)
AI报告缓存: 81.2% (39/48)
===============
```

### 4. 关键性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 平均渲染时间 | < 50ms | UI刷新速度 |
| 平均AI决策时间 | < 300ms | AI思考速度 |
| 向听缓存命中率 | > 90% | 向听数计算效率 |
| 进张缓存命中率 | > 70% | 进张分析效率 |
| AI报告缓存命中率 | > 80% | 完整决策缓存效率 |

## 详细测试场景

### 场景1：快速出牌测试
1. 进入单机模式
2. 连续快速出牌10次
3. 观察：界面应该流畅刷新，无明显卡顿

### 场景2：AI连续行牌测试
1. 等待AI连续出牌
2. 观察：三家AI依次出牌应该流畅，有节奏感
3. 预期：每次AI出牌间隔约0.3-0.5秒（包括动画）

### 场景3：复杂局面测试
1. 进行到中后期（手牌较多变化）
2. 测试吃碰杠操作
3. 观察：复杂决策下UI仍然流畅

### 场景4：内存测试
1. 连续玩8局完整对局
2. 观察内存占用是否稳定
3. 预期：内存增长 < 5MB

## 已知问题排查

### 问题1：AI推荐不显示
**现象：** 首次渲染后AI推荐标记延迟出现  
**原因：** 异步计算优化，属于正常现象  
**影响：** 极小，通常1-2帧后即显示

### 问题2：缓存命中率低
**现象：** 缓存命中率 < 预期值  
**排查：**
1. 检查是否游戏刚开始（缓存还在预热）
2. 检查是否局面变化太快（每次都是新状态）
3. 多玩几局后缓存命中率应该上升

### 问题3：性能提升不明显
**排查：**
1. 确认修改已保存并重新加载
2. 检查是否在快速模式
3. 观察控制台是否有错误信息

## 性能基准参考

基于 Godot 4.6.3 / Android 设备：

**优化前：**
- UI渲染: 500-800ms (阻塞)
- AI决策: 500-800ms
- 总响应延迟: 1000-1600ms

**优化后（预期）：**
- UI渲染: 10-50ms (非阻塞)
- AI决策: 150-250ms (异步)
- 总响应延迟: 10-50ms (UI部分)

**提升：** 10-15倍响应速度提升
